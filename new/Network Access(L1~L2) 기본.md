
OSI 기준 3계층이 아니라 1~2 계층을 다룸.

물리적인 변환 과정이나 그걸 Host에 전기적, 물리적 신호로 보내는거, 연결하는거 (NIC) 등을 다룸.

이더넷 같은게 여기 포함.

따라서 IP를 모름. 대신 Mac 주소를 통해서 통신한다.

그래서 서브 그룹 단위로는 안되고, 보낼 수 있는 전부 or 특정 MAC에 한번씩 만 가능함.

인프라, 백본 관련 내용은 별도로 분리하는게 좋을듯? 라우팅이나 MPLS이나 VLAN이나...

일단 책 내용 키워드 위주로 정리 -> 그 외 기타 자료로 추가 보완 -> 책 다시 보고 정리 -> 위치 나누기 등...

너무 딥하게는 하지 말고, 결론 위주로, 그런건 책 보면 되니까.

- 6.1.2 링크 계층이 구현되는 위치
    - 호스트에서 대부분의 경우 링크 계층은 네트워크 인터페이스 컨트롤러(network interface controller - 요즘엔 card라고 더 많이 부르는듯?, NIC)로 알려진 네트워크 어댑터(network adapter) 에 구현된다.
    - NIC는 어떻게 통신하는가? 
        - 책: controller(물리 -> 전기 신호 변환) -(데이터버스를 통해서)> CPU(인터럽트 발생)
        - 따로 찾은거:  
            - 현대 OS는 다른 하드웨어를 다루는 것처럼 똑같이 다룬다. 
            - NIC는 이더넷 카드, 무선 LAN 어댑터 등 다양한 형태로 제공되며, 최근에는 NIC가 메인보드에 통합된 형태로도 많이 사용된다.
            - NIC는 하드웨어 기기이다. 이에 호환되는 드라이버가 있고, 드라이버는 OS 규칙에 맞게 인터럽트를 발생시킨다.
            - 즉 NIC -> 드라이버 -> 인터럽트 발생([MSI](https://en.wikipedia.org/wiki/Message_Signaled_Interrupts), [DMA](https://en.wikipedia.org/wiki/Direct_memory_access) 등 드라이버 구현따라 다름) -> CPU가 다른 인터럽트와 마찬가지로 동일하게 처리
            - USB to RJ45 어댑터 같이 노트북에 랜 끼우는것도 NIC의 일종으로 볼 수 있음.
            - NIC 하드웨어는 인터럽트만 외에도 링크 계층의 기능인 패킷 버퍼링(Buffering), 에러 검출(Error Detection), 흐름 제어(Flow Control) 
            - NIC가 하는 일
                - 네트워크 매체(예: 이더넷 케이블, 무선 신호)와의 물리적 상호작용. 
                - 물리적 <-> 전기적 신호 변환 
                - 물리적 인터페이스 (RJ45, Wi-Fi 어댑터) 제공
            - NIC 동작
                - (1) 물리적 신호를 프레임으로 변환
                - (2) 에러 검출 및 처리 
                - (3) 흐름 제어 및 버퍼링 수행. 
                - 이후 인터럽트를 발생시켜 OS에게 프레임 정보를 넘긴다.
            - ref: 
                - https://en.wikipedia.org/wiki/Network_interface_controller : 더 찾아봐야함. 부족한 부분도 많음.
                - https://www.geeksforgeeks.org/nic-full-form/

- 6.2 오류 검출 및 정정 기술
    - 패리티 검사 : 신뢰성이 낮아 현대 네트워크에서는 거의 사용되지 않음.
    - 체크섬 : TCP, UDP에서 데이터그램의 무결성을 확인하기 위해 사용. 소프트웨어로 구현됨.
    - 순환 중복 검사(CRC) : 하드웨어에서 효율적으로 구현 가능. 링크 계층(NIC)에서, 물리 계층(무선 통신, UDB, HDD, SSD) 등에서 사용됨.
    - 순서대로 더 높은 오류 검출 능력.
- 6.3 다중 접속 링크와 프로토콜: 다수의 노드가 하나의 링크를 공유하는 경우 어떻게 되는가?
    - [메모] 아래는 책 내용임. 근데 이게 실제로 어디 쓰이는건지? 2계층이라곤 하는데, , 거의 안쓰인다 하지 않았나? 어느 계층 동작인거지. 그리고 내가 이걸 알아야 하나? 필요없어보이는건 키워드만 적어놓기
    - 네트워크 링크의 두 종류
        - 점대점 링크 (Point-to-Point Link)
        - 브로드캐스트 링크 (Broadcast Link)
    - 다중 접속 문제: 여러 노드가 동시에 데이터 전송 시 충돌 발생 → 데이터 손실(의미 파악 불가) 및 채널 낭비.
    - 다중 접속 프로토콜으로 바람직한 특성 - 아래와 같은 특징을 가져야 좋음.
        - 단일 노드 데이터 전송 시 R bps 처리율.
        - N개 노드 동시 전송 시 각 노드는 R/N bps 처리율.
        - 분산형 설계(마스터 노드 필요 없음), 단순함.
    - 채널 분할 프로토콜
        - TDM (시분할 다중화):
            - 시간을 프레임 및 슬롯으로 나눔. 각 노드가 자신의 시간 슬롯에만 데이터 전송.
            - 충돌 제거, 공정하지만, 노드의 수와 상관 없이 항상 R/N이므로 비효율적임.
            - e.g. 위성 통신.
        - FDM (주파수 분할 다중화):
            - 채널을 주파수 대역으로 나눠 각 노드에 할당.
            - TDM과 유사한 장단점.
            - e.g. 라디오 방송, 케이블 TV.
        - CDMA (코드 분할 다중 접속):
            - 각 노드에 고유 코드 할당. 여러 노드가 동시에 전송 가능.
    - 랜덤 접속 프로토콜
        - 충돌 발생 시 프레임 재전송. 랜덤 지연 시간으로 충돌 완화.
        - 슬롯 알로하: 슬롯 단위로 전송. 최대 효율 37%. 슬롯을 사용하므로 동기화 되어야 함.
        - 순수 알로하: 슬롯 없이 전송. 최대 효율 18%(슬롯 알로하의 절반). 대신 동기화가 필요하지 않음.
        - CSMA: 다른 노드가 전송 중인지 확인 후 전송.
        - CSMA/CD: 충돌 감지 시 전송 중단, 지연 후 재시도. 
    - 순번 프로토콜
        - 폴링 프로토콜: 마스터 노드가 각 노드 순차적으로 관리.
        - 토큰 전달 프로토콜: 노드 간 토큰을 전달하며 전송 기회 제공.
    - DOCSIS (케이블 인터넷 프로토콜 - 케이블 인터넷 접속을 위한 링크 계층 프로토콜)
        - 하향 채널: FDM 기반, 충돌 없음, CMTS가 데이터 송신 제어.
        - 상향 채널: CMTS와 케이블 모뎀이 미니슬롯 기반으로 데이터 송신.
        - 충돌 시 이진 지수적 백오프(Exponential Backoff - 재시도 간격을 점진적으로 늘려가며 재시도) 적용.
- 6.4 스위치 근거리 네트워크
    - 6.4.1 링크 계층 주소체계와 ARP
        - 링크 계층 주소체계: 네트워크 계층 주소 체계가 있는데도 링크 계층 주소를 갖는 이유
            - LAN은 임의의 네트워크 계층 프로토콜을 위해 설계되었기 때문. 
            - 각 계층이 독립적이기 위해선 자신만의 주소 기법을 가져야 함.
            - [메모] 번외로 MAC과 IP 중 하나만 사용해도 네트워크는 잘 동작할 텐데, 분리하는 이유를 궁금해 했던적이 있는데, 저런 네트워크 계층 이유를 배제하도고 다른 이유는 IP는 논리적인 연관관계 (서브넷 같은거)가 있는 반면, MAC은 없어서 불가능하다. 
        - MAC: 48bit, 6byte, 앞 절반은 IEEE에서 제조사에게 할당. 나머지는 제조사가 알아서 중복이 생기지 않게 할당.
        - 송수신: 송신 어댑터는 본인의 MAC과 요청에 온 MAC이 일치하는지 (or 브로드캐스트인지) 확인하고 아니라면 폐기, 이외에는 상위 스택으로 전달
        - ARP: IP를 보고 목적 MAC 주소를 구하는 프로토콜
            - 각 호스트와 라우터는 자신의 메모리에 ARP 테이블(ARP table)을 가진다. - IP주소와 대응되는 MAC 주소, TTL 값을 포함.
                - ARP 테이블은 플러그 앤 플레이(plug-and-play)이다. 알아서 동작하면서 자동으로 구축됨.
            - ARP 호출 시 목적 IP값을 담아서 브로드캐스트로 요청, 각 노드는 요청을 받고, 본인이 해당 IP를 가지는 경우 응답함. 호출자는 응답을 통해서 IP와 매핑되는 MAC주소를 알게 되므로 테이블에 적음. 테이블에 있으면 요청을 하지 않고, 바로 호출함.
        - [매모] RARP는 IP로 MAC을 확인하는거, 저장장치가 없는 경우 인접 노드로부터 정보를 구하는 용도로 썼다는데, 요즘에는 거의 DHCP랑 따른거로 대체되었다고 함.
    - 6.4.2 이더넷 : 근거리 네트워킹을 다룸. IEEE 표준
        1. 이더넷 프레임 구조
            - 데이터 필드(46~1500바이트): IP 데이터그램 운반.
            - 목적지/출발지 MAC 주소(각 6바이트).
            - 프리앰블(8바이트): 수신기 클록 동기화.
            - 순환 중복 검사(CRC, 4바이트): 오류 검출.
        2. 서비스 특성
            - 비연결형 서비스: 핸드셰이킹 없이 전송.
            - 비신뢰적 서비스: 프레임 손실 시 확인 응답 없음. 손실 시 상위 계층(TCP 등)이 처리.
        3. 현대 이더넷
            - 스타 토폴로지: 스위치 기반. 충돌 없는 점대점 연결.
            - CSMA/CD 프로토콜 불필요.
    - 6.4.3 링크 계층 스위치
        1. 스위치의 역할
            - 필터링: 전송 불필요한 프레임은 폐기.
            - 포워딩: 프레임을 적절한 출력 인터페이스로 전송.
        - 2. 스위치 테이블
            - 구성: MAC 주소, 해당 인터페이스, 생성 시간.
            - 작동 방식:
                - 엔트리 없음: 브로드캐스트.
                - 현재 인터페이스와 연관된 엔트리: 폐기(필터링).
                - 다른 인터페이스와 연관된 엔트리: 해당 인터페이스로 전달(포워딩).
        - 스위치의 자가학습(Self-Learning)
            - 동적으로 스위치 테이블 생성 - plug and play
            - 수신 프레임 기반 학습: 
                - 스위치 처음 상태 = 빈 테이블
                - (스위치가) 프레임 수신할 때마다 자동 기록 
                - 어떤 MAC 주소에서 '수명시간(aging time)' 동안 데이터가 오지 않으면 삭제
                - 네트워크의 모든 기기가 통신하면서 테이블 자동 완성
        - 링크 계층 스위치의 특성
            - 충돌 제거: 프레임을 버퍼링하여 한 세그멘트(간단하게 포트라고 보면 됨)에는 하나의 프레임만 전송
            - 이질적인 링크 지원: 서로 다른 매체나 속도를 사용하는 링크를 연결 가능.
            - 관리 용이성: 네트워크 관리와 문제 감지 기능 제공. - 아마 그 CCNA에서 말하는 그런거? - 그외에도 그냥 기기로써의 util 같은 부분
            - 보안: 스위치 테이블에 엔트리가 있을 때, 한 호스트로만 전송하므로, 다른 호스트는 홈쳐볼 수 없다.
                - 그러나 스위치 테이블에 엔트리가 없을 때, 브로드캐스트하기 때문에, 일부 프레임을 훔쳐볼 수 있다. - 스위치 posioning 이라는 기법
                - [메모] 허브의 경우 무조건 다 보내니까 스니핑이 계속 가능함.
        - 스위치 대 라우터
            - 아래 AI 요약본 확인
            - CCNA 추가로 검토하기. 근데 위 스위치 문제점도 다 CCNA 쪽에서 해결책 다뤘던거 같음. 근데 그것도 라우터 함께 써서 그랬던거 같기도 하고...
    - 6.4.4 가상 근거리 네트워크(VLAN)
        - 기존 계층형 스위치 구조 단점: 
            - 트래픽 격리 부족:
                - 브로드캐스트 트래픽이 전체 네트워크에 전달됨.
            - 스위치 비효율:
                - 더 많은 그룹이 필요할 경우 다수의 스위치 추가 필요.
            - 사용자 관리의 번거로움:
                - 그룹 이동 시 물리적 케이블 재배치 필요.
        - VLAN의 동작
            - 하나의 물리적 네트워크 인프라에서 여러 개의 가상 네트워크(VLAN)를 정의 가능.
            - 포트 기반 VLAN: 스위치의 포트를 그룹으로 나누어 VLAN을 형성.
                - 각 VLAN은 독립된 브로드캐스트 도메인을 가짐.
        - VLAN의 구현
            - 포트-VLAN 매핑 테이블 관리
                - 관리자는 VLAN에 속한 포트를 지정하고 VLAN 간 격리를 유지.
                - 동일 VLAN 내에서만 프레임 전달.
            - VLAN 간 통신
                - VLAN 간 트래픽은 라우터를 통해 전달.
                - VLAN 스위치의 트렁크 포트를 이용하여 VLAN 식별 및 연결.
        - VLAN 태그
            - 확장 이더넷 프레임 형식을 사용하여 VLAN을 식별. - IEEE에서 제공하는 표준 확장임. 
                - VLAN 태그: 송신 스위치에서 추가, 수신 스위치에서 제거.
            - VLAN 태그 구성:
                - 2바이트 TPID (Tag Protocol Identifier)
                - 2바이트 TCI (Tag Control Information)
                    - 12비트 VLAN ID
                    - 3비트 우선순위 필드
        - VLAN 트렁킹
            - 트렁크 포트를 통해 여러 VLAN 연결.
            - IEEE 표준에 따라 VLAN 프레임에 태그를 추가하여 VLAN 구분.
            - 확장성 문제를 해결하고 VLAN 간 통신을 효율적으로 처리.
    - 6.5 링크 가상화: 링크 계층으로서의 네트워크 - 백본에서 사용되는 MPLS
        - > MPLS의 목표는 고정 길이 레이블과 가상 회선을 기반으로 데이터그램을 전달하기 위해 목적지 기반 IP 데이터그램 인프라스트럭처를 포기하는 것이 아니라, 가능한 경우 데이터그램을 선택적으로 레이블링해서 라우터로 하여금 고정 길이 레이블을 기반으로 데이터그램을 전달할 수 있도록 목적지 기반 IP 데이터그램 전달 하부구조를 확장하는 것이다.

## 물리적 네트워킹

이 모든 네트워크 데이터는 구리, 광섬유, 무선 라디오와 같은 물리적 매체를 통해 전송되어야 합니다. 물리 계층 프로토콜 중에서 이더넷(Ethernet)이 가장 잘 알려져 있습니다. 인터넷 초기 시절 이더넷의 인기는 그 한계를 보완하기 위해 다른 프로토콜을 설계하게 만든 계기가 되었습니다.

먼저 물리적인 세부 사항을 살펴보겠습니다. 이더넷은 주로 RJ45 커넥터와 연관이 있으며, 이는 구형 4핀 전화 잭보다 더 큰 8핀 버전처럼 보입니다. 또한 8개의 전선을 4개의 꼬임 쌍으로 구성한 cat5(또는 cat5e, cat6, cat7) 케이블과도 관련이 있습니다. 다른 매체도 존재하지만, 우리가 집에서 가장 흔히 접할 수 있는 것은 바로 이러한 8개의 전선이 덮개에 감싸여 8핀 잭에 연결된 형태입니다.

이더넷은 물리 계층 프로토콜로, 케이블 내에서 비트를 전기 신호로 변환하는 방식을 설명합니다. 또한 링크 계층 프로토콜로서 한 노드에서 다른 노드로의 직접 연결 방식을 설명합니다. 하지만 이더넷은 순수한 점대점 방식으로 작동하며 네트워크에서 데이터를 라우팅하는 방식에 대해서는 언급하지 않습니다. TCP 연결의 의미에서의 연결 개념이나 IP 주소의 재할당 가능성과 같은 기능은 없습니다.

### 이더넷의 두 가지 주요 역할
이더넷은 프로토콜로서 두 가지 주요 역할을 수행합니다. 첫째, 각 장치는 다른 장치와 연결되었음을 인지해야 하며, 연결 속도와 같은 몇 가지 매개변수를 협상해야 합니다.  
둘째, 링크가 설정된 후 이더넷은 데이터를 전송해야 합니다. TCP 및 IP와 같은 상위 계층 프로토콜과 마찬가지로 이더넷 데이터는 패킷으로 나뉩니다. 패킷의 핵심은 프레임으로, 1,500바이트의 페이로드와 소스 및 목적지 MAC 주소, 페이로드 길이, 체크섬 등 헤더 정보로 22바이트를 추가로 포함합니다. 프로그래머는 주소, 길이, 체크섬과 같은 필드에 익숙하며, 이들이 왜 필요한지 쉽게 이해할 수 있습니다.

프레임은 또 다른 헤더 계층으로 감싸져 전체 패킷이 됩니다. 이 헤더는 독특합니다. 이는 아날로그 전기 시스템의 본질적인 특성과 맞닿아 있어 소프트웨어 프로토콜에서 볼 수 없는 형식으로 나타납니다. 전체 이더넷 패킷은 다음을 포함합니다:

1. **프리앰블**: 56비트(7바이트)의 1과 0이 교대로 나타납니다. 장치는 이를 사용하여 시계를 동기화하며, 이는 사람들이 "하나, 둘, 셋, 시작!"이라고 외치는 것과 비슷합니다. 컴퓨터는 1 이상을 셀 수 없으므로 "101010...10"을 반복하여 동기화를 맞춥니다.
2. **시작 프레임 구분자(SFD)**: 8비트(1바이트) 값으로 171(이진수로 10101011)입니다. 이는 프리앰블의 끝을 표시합니다.
3. **프레임 본체**: 소스 및 목적지 주소, 페이로드 등으로 구성됩니다.
4. **패킷 간 간격**: 96비트(12바이트)로, 라인은 이 기간 동안 유휴 상태가 됩니다. 이는 장치가 잠시 휴식할 시간을 제공하기 위한 것으로 보입니다.

결론적으로, 1,500바이트의 데이터를 전송하려면 22바이트를 추가하여 프레임을 생성하고, 소스, 목적지, 크기 및 체크섬을 나타냅니다. 그런 다음 하드웨어의 요구를 충족하기 위해 20바이트를 추가로 더해 전체 이더넷 패킷을 만듭니다.

- [메모] TCP/UDP는 16비트의 길이 필드를 가지므로 65,535바이트까지 표현 가능하다. (헤더 포함 전체 크기가) 그러나 실제 사용시에는 대부분 이더넷의 MTU(최대 크기)인 1500 바이트 이하로 보낸다고 함.
    - 여러 이유가 있는거 같긴 함. TCP 단에서 큐에 오래 있어야 한다거나... 여러 패킷을 묶어서 하나로 하니까 에러 발생 확률이 높아진다거나? 하드웨어 구현이 이에 맞춰있다거나? 뭐... - 근데 이걸 깊게 찾아볼 필요는 없어보이고.

### 네트워킹과 현실 세계의 만남
디지털 시스템은 실제로 존재하지 않습니다. 모든 것은 아날로그입니다.  
5볼트 CMOS 시스템을 예로 들어 보겠습니다. 완전히 켜진 신호는 5볼트, 꺼진 신호는 0볼트입니다. 그러나 실제로는 1.67볼트 이상을 1로, 1.67볼트 이하를 0으로 간주합니다. 이더넷 패킷이 물리적 와이어를 통해 전송될 때 전압을 변경해야 합니다. 또한 8비트 데이터는 10비트로 확장되어야 하며, 이는 전기적 균형을 맞추기 위한 것입니다.

### 프로토콜 계층의 독립성
인터넷 프로토콜은 계층화된 스택으로 생각하는 것이 가장 좋습니다. 이더넷은 물리적 데이터 전송 및 점대점 장치 간의 링크를 제공합니다. IP는 라우터 및 대규모 네트워크를 허용하는 주소 계층을 제공하지만, 연결 지향적이지는 않습니다. TCP는 시퀀스 번호, 확인 응답, 재전송을 통해 신뢰할 수 있는 전송 계층을 추가합니다. HTTP와 같은 애플리케이션 레벨 프로토콜은 TCP 위에 계층화됩니다.

프로토콜 계층의 독립성은 중요한 설계 원칙입니다. 예를 들어, 데이터가 손실되더라도 이를 감지하고 해결하는 것은 TCP의 역할이며, 네트워크의 다른 계층은 이를 알 필요가 없습니다.

결론적으로, 현대 프로토콜 설계는 과거 하드웨어 제약에서 비롯된 선택의 결과입니다. 이더넷 페이로드 크기가 1,500바이트로 설정된 이유는 효율성과 메모리 사용 간의 균형을 맞추기 위해서였습니다. HTTP/2의 헤더 압축은 1970년대 후반 네트워킹 장치의 RAM 한계에서 시작된 설계 제약의 결과입니다.

1계층(물리 계층)은 데이터를 전송하기 위한 물리적 매체와 신호를 정의하는 계층입니다. 물리 계층에서는 주로 프로토콜보다는 **물리적 특성, 신호 처리, 전송 매체**에 관한 표준이 중심이 됩니다. 다만, 물리 계층의 전송을 지원하기 위해 특정 프로토콜과 표준이 사용되며, 주요 요소는 다음과 같습니다.

## **1. 물리 계층에서 사용되는 주요 프로토콜과 표준**
물리 계층은 하드웨어와 관련된 사항을 정의하므로 특정 "프로토콜"보다는 **표준화된 기술**을 사용하는 경우가 많습니다.

### **대표적인 물리 계층 표준**
1. **이더넷 표준 (IEEE 802.3)**  
   - 데이터 링크 계층과 연계된 물리 계층 표준을 포함합니다.  
   - 속도 및 매체에 따라 여러 세부 표준이 존재:
     - Fast Ethernet (100Base-T)
     - Gigabit Ethernet (1000Base-T)
     - 10-Gigabit Ethernet (10GBase-T)
   - 매체의 유형(동축 케이블, 꼬임쌍선, 광섬유)과 전송 속도를 정의.

2. **Wi-Fi (IEEE 802.11 시리즈)**  
   - 무선 통신을 위한 표준.
   - 물리 계층에서 신호의 변조 방식과 주파수 대역 정의.
   - ex) 802.11a, 802.11b, 802.11g, 802.11ac 등.

3. **블루투스 (IEEE 802.15.1)**  
   - 근거리 무선 통신 표준.
   - 신호의 주파수 홉핑 방식과 전송 속도를 정의.

4. **USB (Universal Serial Bus)**  
   - 물리적인 연결과 데이터 전송 방식에 대한 표준.

5. **DSL (Digital Subscriber Line)**  
   - 전화선 기반의 데이터 전송 기술.

6. **광섬유 표준 (ITU-T G 시리즈)**  
   - G.652 (표준 단일 모드 광섬유)  
   - G.709 (광 전송망 표준)

## **2. 물리 계층에서 정의되는 값**
1계층에서는 신호와 전송 매체의 특성을 정의하므로, 사용되는 값들은 데이터가 신호로 변환되는 방식이나 매체 특성과 관련됩니다.

### **주요 물리적 특성 값**
1. **전송 속도 (Transmission Rate)**  
   - 초당 전송 가능한 비트 수 (bps).  
   - 예: 10Mbps, 1Gbps, 10Gbps 등.

2. **신호 레벨 (Signal Levels)**  
   - 디지털 신호에서 사용할 전압 수준이나 광 신호 강도.

3. **주파수 대역 (Frequency Band)**  
   - 사용하는 주파수 범위.  
   - 예: Wi-Fi의 경우 2.4GHz, 5GHz 대역.

4. **변조 방식 (Modulation Techniques)**  
   - 데이터를 신호로 변환하는 방법.
     - ASK (Amplitude Shift Keying)
     - FSK (Frequency Shift Keying)
     - PSK (Phase Shift Keying)
     - OFDM (Orthogonal Frequency Division Multiplexing)

5. **케이블 특성**  
   - 케이블의 길이 제한, 저항, 대역폭.
   - 예: CAT5e 케이블은 최대 100m 거리에서 1Gbps 속도 지원.

6. **전송 매체 유형**  
   - 매체의 특성:
     - 꼬임쌍선(Twisted Pair)
     - 동축 케이블(Coaxial Cable)
     - 광섬유(Fiber Optic)
     - 무선(Wireless)

7. **동기화 (Synchronization)**  
   - 송신자와 수신자가 신호 타이밍을 일치시키기 위한 클록 신호.

### **결론**
1계층은 데이터의 전송과 수신을 물리적으로 구현하는 계층으로, **프로토콜보다는 표준과 물리적 특성**(전송 속도, 주파수, 변조 방식 등)이 중심이 됩니다.  
데이터 링크 계층의 프레임이 물리 계층에서는 전기 신호, 광 신호, 또는 무선 신호로 변환되어 전송되므로, 물리 계층의 설정이 데이터 전송의 성능과 신뢰성을 결정합니다.

물리 계층 프로토콜(Layer 1)과 관련된 주요 내용을 다음과 같이 정리할 수 있습니다:

## 전송 매체와 신호 전파

**광섬유 케이블**
- 광섬유는 머리카락보다 약간 두꺼운 "빛의 파이프" 역할을 합니다[1].
- 금속 와이어에 비해 신호 손실이 적고 전자기 간섭에 강합니다[1].
- 장거리 전송에 주로 사용되며, 대륙 간 해저 데이터 전송에도 활용됩니다[1].

**전파 속도**
- 진공에서 빛의 속도는 299,792,458 m/s입니다[1].
- 광섬유에서는 굴절률로 인해 속도가 약 200,000,000 m/s로 감소합니다[1].
- 광섬유의 일반적인 굴절률은 1.4에서 1.6 사이입니다[1].

## 대역폭 (Bandwidth)

**광섬유의 대역폭**
- 파장 분할 다중화(WDM) 기술을 통해 하나의 광섬유에서 여러 채널을 전송할 수 있습니다[1].
- 2010년 초 기준으로 400개 이상의 파장을 다중화할 수 있으며, 채널당 최대 171 Gbit/s의 용량을 가집니다[1].
- 단일 광섬유 링크의 총 대역폭은 70 Tbit/s 이상에 달할 수 있습니다[1].

**백본 네트워크**
- 장거리 전송에는 주로 광섬유 케이블이 사용됩니다[1].
- 각 케이블은 여러 가닥의 광섬유를 포함하며, 일반적으로 4가닥이 사용됩니다[1].
- 이는 각 케이블에 대해 수백 테라비트/초의 대역폭 용량을 제공합니다[1].

## 지연 시간 (Latency)

**전파 지연**
- 신호가 송신자에서 수신자로 이동하는 데 필요한 시간으로, 거리와 신호 전파 속도의 함수입니다[1].

**전송 지연**
- 패킷의 모든 비트를 링크에 밀어 넣는 데 필요한 시간으로, 패킷의 길이와 링크의 데이터 속도의 함수입니다[1].

**라스트 마일 지연**
- 최종 사용자와 인터넷 서비스 제공업체(ISP) 사이의 연결에서 발생하는 지연입니다[1].
- 미국의 경우, 광섬유(10-20 ms), 케이블(15-40 ms), DSL(30-65 ms)의 순으로 성능이 좋습니다[1].

물리 계층은 네트워크 성능에 직접적인 영향을 미치며, 특히 지연 시간과 대역폭은 사용자 경험에 중요한 역할을 합니다. 따라서 네트워크 설계 시 이러한 요소들을 신중히 고려해야 합니다.

### **스위치 vs 라우터 요약**

#### **스위치**
- **역할**: 2계층에서 **MAC 주소**를 기반으로 프레임 전달.  
- **특징**:  
  - 충돌 도메인 분리, 네트워크 성능 향상.  
  - **플러그 앤 플레이**로 설치와 관리가 간단.  
  - 브로드캐스트 프레임 순환 방지를 위해 **스패닝 트리**로 토폴로지 제한.  
  - 대규모 네트워크에서 ARP 트래픽 증가, 브로드캐스트 폭주 대책 부족.  

#### **라우터**
- **역할**: 3계층에서 **IP 주소**를 기반으로 패킷 전달.  
- **특징**:  
  - 충돌 및 브로드캐스트 도메인 분리 가능.  
  - 최적 경로 설정으로 네트워크 확장성 제공.  
  - 브로드캐스트 폭풍 방지 및 방화벽 기능 포함.  
  - 설치와 관리 복잡, **플러그 앤 플레이**가 아님.  

#### **사용 상황**
- **소규모 네트워크**: 스위치만으로 충분.  
- **대규모 네트워크**: 스위치와 라우터 병행 사용.

이더넷(Ethernet)은 **OSI 7계층 모델에서 주로 1계층(물리 계층, Physical Layer)**과 **2계층(데이터 링크 계층, Data Link Layer)**에 속합니다. 

### **이더넷이 OSI 모델의 1계층과 2계층에 속하는 이유**  

---

### **1. 물리 계층 (Physical Layer)**  
- **역할**: 데이터가 실제로 물리적 매체(케이블, 광섬유 등)를 통해 전송되도록 신호를 변환합니다.  
- **이더넷 관련 기능**:  
  - **전기적 신호 변환**: 데이터를 0과 1의 전기적 신호로 변환하여 케이블로 전송.  
  - **물리적 연결**: RJ45 커넥터, Cat5/Cat6/Cat7 케이블과 같은 물리적 매체를 정의.  
  - **전송 속도**: 전송 속도(10 Mbps, 1 Gbps 등)를 물리적으로 구현.  

#### **이더넷이 물리 계층에 속하는 이유**:  
이더넷 표준(IEEE 802.3)은 데이터를 전기 신호로 변환하고 이를 물리적 매체로 전송하는 방식을 정의합니다. 이는 물리 계층의 주요 기능과 일치합니다.

---

### **2. 데이터 링크 계층 (Data Link Layer)**  
- **역할**: 네트워크에서 바로 연결된 두 장치 간 데이터 전송을 제어하고 오류를 처리합니다.  
- **이더넷 관련 기능**:  
  - **MAC 주소**: 각 네트워크 인터페이스에 고유한 MAC 주소를 부여하여 데이터 전송 대상 식별.  
  - **프레임 생성**: 데이터 전송을 위한 프레임 구조를 정의(헤더, 페이로드, 트레일러 포함).  
  - **에러 검출**: 체크섬(CRC)으로 데이터 무결성을 확인.  
  - **CSMA/CD**: 충돌 감지 및 대기 시간을 조정해 네트워크 충돌 관리(구형 이더넷).  

#### **이더넷이 데이터 링크 계층에 속하는 이유**:  
이더넷은 데이터 프레임 생성, MAC 주소를 통한 장치 식별, 충돌 관리 등의 기능을 수행하며, 이는 데이터 링크 계층의 핵심 역할에 해당합니다.

---

### **결론: 이더넷은 OSI 모델에서 1계층과 2계층 모두를 포함**  
- **물리 계층(1계층)**: 전기적/광학적 신호로 데이터 전송을 담당.  
- **데이터 링크 계층(2계층)**: MAC 주소를 기반으로 데이터 프레임 전송과 오류 관리를 수행.  

이를 통해 이더넷은 하위 네트워크 계층에서 데이터가 안정적으로 전송되고 물리적 매체를 통해 전달되도록 지원합니다.

--------------------------

OSI 모델의 1계층(물리 계층)과 2계층(데이터 링크 계층)은 이론적으로 구분되어 있지만, 실제 구현에서는 상당히 밀접하게 연관되어 있습니다. 이는 특히 네트워크 하드웨어와 물리적 통신 매체에서 두 계층의 기능이 상호의존적이기 때문입니다. 아래에서 이를 더 상세히 설명합니다.

---

### **1계층: 물리 계층 (Physical Layer)**
- **역할**: 데이터를 물리적 신호로 변환하여 전송하는 것을 담당.
  - 전기적 신호, 광 신호, 라디오 신호 등의 **매체별 변환과 전송**.
  - 케이블, 커넥터, 전송 매체의 스펙(전압, 주파수, 대역폭 등) 정의.
- **핵심 기술**:
  - **신호 인코딩 및 변조**: 디지털 데이터를 아날로그 신호로 변환(예: NRZ, PSK, QAM).
  - **동기화**: 송신자와 수신자의 클럭 신호 동기화.
  - **전송 속도**: 초당 비트 전송률 결정.
  - **물리적 커넥터**와 표준 정의(예: RJ45, USB).
  
---

### **2계층: 데이터 링크 계층 (Data Link Layer)**
- **역할**: 물리 계층에서 받은 신호를 데이터로 처리하고, 오류를 감지하거나 복구하며 데이터의 신뢰성을 보장.
  - 네트워크 노드 간의 **프레임(Frame)** 단위 데이터 전송 관리.
  - **에러 검출/복구**(예: CRC, ARQ).
  - **매체 접근 제어(MAC)** 프로토콜: 여러 장치가 동일한 물리 매체를 공유하는 경우 충돌 방지(예: CSMA/CD, CSMA/CA).
- **핵심 기술**:
  - **주소화**: 네트워크의 각 장치를 식별하기 위해 MAC 주소 사용.
  - **프레임 생성**: 데이터에 헤더와 트레일러를 추가하여 오류 감지 정보 포함.
  - **흐름 제어** 및 **혼잡 제어**: 송수신 속도 차이 문제 해결.

---

### **1계층과 2계층의 상호 의존성**
1. **전기적 신호와 논리적 데이터의 상호 변환**:
   - 1계층에서 전송된 전기적 신호(아날로그 신호)는 2계층에서 디지털 데이터로 변환되어야 함.
   - 디지털 데이터를 다시 전기적 신호로 변환하는 과정에서 신호 인코딩/디코딩 기법(예: Manchester Encoding) 필요.
   
2. **에러 검출 및 복구**:
   - 1계층에서 신호 감쇠, 간섭 등이 발생하면 데이터 손상이 생길 수 있음.
   - 2계층에서 이를 감지하고 복구하거나 재전송 요청을 수행(예: 체크섬 사용).

3. **전송 매체와 MAC 프로토콜**:
   - 유선 LAN(예: Ethernet)이나 무선 LAN(예: Wi-Fi)의 경우, 2계층의 MAC 프로토콜은 1계층의 물리 매체 특성을 이해하고 충돌 관리.

4. **동기화와 흐름 제어**:
   - 물리 계층에서 클럭 신호 동기화 실패는 2계층 데이터 전송의 실패로 이어짐.
   - 2계층은 이를 보완하기 위한 흐름 제어 알고리즘(예: 슬라이딩 윈도우)을 제공.

---

### **결론**
OSI 모델은 설계와 개념 이해를 위해 계층적으로 나뉘어 있지만, 물리적 신호를 다루는 1계층과 데이터의 신뢰성을 보장하는 2계층은 실제 네트워크 환경에서 긴밀히 협력합니다. 따라서, 네트워크 설계자와 엔지니어는 두 계층의 문제를 함께 이해하고 고려해야 안정적이고 신뢰할 수 있는 데이터 전송 시스템을 구현할 수 있습니다.

-------------------------

네, **MAC(Media Access Control) 프로토콜**은 유선 LAN(Ethernet)과 무선 LAN(Wi-Fi)에서 각각 다른 물리 매체의 특성에 맞춰 동작 방식을 달리합니다. 이는 유선과 무선 환경에서 데이터 충돌(Collision) 발생 조건과 그 관리 방법이 다르기 때문입니다.

---

### **유선 LAN(Ethernet)에서의 MAC 프로토콜**
- **특성**: 유선 네트워크는 물리적 케이블을 통해 데이터가 전송되며, 충돌이 발생할 가능성이 무선보다 낮습니다. 하지만 다수의 장치가 동일한 케이블(공유 매체)을 사용할 경우 충돌 가능성이 존재합니다.
- **사용되는 MAC 프로토콜**: 
  - **CSMA/CD (Carrier Sense Multiple Access with Collision Detection)**.
  - **동작 원리**:
    1. 송신 전, 네트워크 매체 상태를 확인(Carrier Sense).
    2. 매체가 비어 있으면 데이터를 전송.
    3. 충돌이 감지되면(Collision Detection) 전송을 중단하고, 충돌 신호를 브로드캐스트로 알림.
    4. 충돌 이후, 임의의 시간만큼 대기(Backoff)하고 재전송.
  - **충돌 관리**: 케이블 기반 환경에서는 충돌을 감지할 수 있기 때문에, 이를 효율적으로 처리하는 CD 메커니즘을 사용.

---

### **무선 LAN(Wi-Fi)에서의 MAC 프로토콜**
- **특성**: 무선 환경에서는 신호가 공중을 통해 전송되므로, 물리적으로 동일 매체를 점유하는 장치 간 충돌뿐 아니라 간섭(Interference) 문제가 추가적으로 발생합니다.
  - 충돌을 감지하기 어렵고, 대신 충돌을 **회피(Avoidance)**하는 방식이 주로 사용됩니다.
- **사용되는 MAC 프로토콜**:
  - **CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)**.
  - **동작 원리**:
    1. 송신 전, 매체 상태를 확인(Carrier Sense).
    2. 매체가 비어 있으면 전송 준비.
    3. 충돌을 회피하기 위해 송신 전에 **RTS/CTS(Request to Send/Clear to Send)** 절차를 사용.
       - 송신자는 "전송해도 되는지" 수신자에게 요청(RTS)하고, 수신자가 허락(CTS)하면 데이터 전송 시작.
    4. 데이터가 성공적으로 전송되면 ACK(확인 메시지) 수신.
    5. ACK을 받지 못하면 충돌이 발생한 것으로 간주하고 재전송.
  - **충돌 관리**: 무선 환경에서는 충돌을 직접 감지하기 어렵기 때문에 충돌이 발생하지 않도록 사전 회피에 초점.

---

### **유선과 무선에서 MAC 프로토콜 차이의 이유**
1. **충돌 감지의 가능 여부**:
   - 유선 LAN에서는 송신 신호와 수신 신호를 비교하여 충돌을 감지할 수 있음.
   - 무선 LAN에서는 동일한 안테나로 송신과 수신을 동시에 수행하기 어렵기 때문에 충돌 감지가 불가능.

2. **공유 매체의 특성**:
   - 유선 네트워크는 제한된 물리적 환경(케이블)을 사용하여 충돌 가능성을 예측하고 관리 가능.
   - 무선 네트워크는 전파가 공간에 퍼지므로 예측하기 어려운 간섭이나 신호 손실 가능성 존재.

3. **전송 거리와 간섭**:
   - 유선 LAN은 고정된 케이블로 연결되므로 전송 거리의 영향을 받지 않음.
   - 무선 LAN은 거리, 장애물, 다른 신호 간섭 등 다양한 요인에 영향을 받음.

---

### **비교 요약**
| 특징                | 유선 LAN (Ethernet)                      | 무선 LAN (Wi-Fi)                      |
|---------------------|-----------------------------------------|---------------------------------------|
| 사용 MAC 프로토콜   | CSMA/CD                                 | CSMA/CA                               |
| 충돌 관리 방법      | 충돌 **감지** 및 재전송                 | 충돌 **회피** 및 재전송               |
| 충돌 발생 확인      | 송신/수신 신호 비교로 충돌 감지 가능     | 충돌 감지 불가능, ACK을 통해 확인     |
| 신호 간섭 문제      | 없음 (물리적 케이블 연결)               | 있음 (공간적 간섭 및 신호 손실)       |

---

### **결론**
유선 LAN과 무선 LAN은 물리 매체의 특성이 다르기 때문에 MAC 프로토콜도 각각의 특성에 맞게 최적화되어 있습니다. 유선은 충돌을 감지하고 대응하는 방식(CD)을 사용하며, 무선은 충돌을 사전에 방지하는 방식(CA)을 사용합니다. 이로 인해 동일한 MAC 계층이라도 유선과 무선에서의 구현 및 동작이 달라질 수밖에 없습니다.
